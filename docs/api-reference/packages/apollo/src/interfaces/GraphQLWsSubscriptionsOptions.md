| <a id="transportidletimeout"></a> `transportIdleTimeout?` | `number` | Transport-level idle timeout, in **seconds**, forwarded to Bun's `ws.idleTimeout`. When the WebSocket is silent for this many seconds in both directions, uWebSockets closes the connection at the transport layer, guaranteeing cleanup even if the application-level watchdog is disabled. Prefer this over `keepAliveTimeout` when possible — it's handled natively and doesn't require a JS timer per connection. Values are in seconds to match Bun's API. When unset, derived from `keepAliveTimeout` (+ 5s grace) so the transport closes shortly after the app-level watchdog would fire. Falls back to Bun's 120s default when neither is set.
