# packages/microservices

## Classes

| Class | Description |
| ------ | ------ |
| [BaseServer](classes/BaseServer.mdx) | Abstract base class for all built-in transport servers. |
| [ClientFactory](classes/ClientFactory.mdx) | Factory that creates the correct [ClientProxy](classes/ClientProxy.mdx) implementation for the given transport configuration. |
| [ClientProxy](classes/ClientProxy.mdx) | Abstract base class for all microservice client proxies. |
| [ElysiaNestApplication](classes/ElysiaNestApplication.mdx) | Top-level application class that ties together an Elysia HTTP server and one or more microservice transport servers. |
| [RabbitMQClient](classes/RabbitMQClient.mdx) | Client proxy that communicates over RabbitMQ (AMQP via amqplib). |
| [RabbitMQServer](classes/RabbitMQServer.mdx) | Transport server that uses RabbitMQ (AMQP via amqplib). |
| [RedisClient](classes/RedisClient.mdx) | Client proxy that communicates over Redis Pub/Sub. |
| [RedisServer](classes/RedisServer.mdx) | Transport server that uses Redis Pub/Sub for message passing. |
| [ServerFactory](classes/ServerFactory.mdx) | Factory that instantiates the correct transport server based on the provided [MicroserviceOptions](type-aliases/MicroserviceOptions.mdx). |
| [TcpClient](classes/TcpClient.mdx) | Client proxy that communicates over raw TCP with newline-delimited JSON framing. |
| [TcpServer](classes/TcpServer.mdx) | Transport server that uses raw TCP sockets with newline-delimited JSON framing for request-response and fire-and-forget communication. |

## Functions

| Function | Description |
| ------ | ------ |
| [Catch](functions/Catch.mdx) | Marks an exception filter class and declares which exception types it handles. Omitting arguments causes the filter to catch **all** exceptions. |
| [Client](functions/Client.mdx) | Marks a class property for automatic [ClientProxy](classes/ClientProxy.mdx) injection. The injected client is created using `config` and is ready to use after the application starts. |
| [createElysiaNestApplication](functions/createElysiaNestApplication.mdx) | Creates a bare [ElysiaNestApplication](classes/ElysiaNestApplication.mdx) that wraps an already-configured Elysia instance. |
| [createElysiaNestApplicationWithControllers](functions/createElysiaNestApplicationWithControllers.mdx) | Creates a new [ElysiaNestApplication](classes/ElysiaNestApplication.mdx) with a set of controllers pre-registered for pattern handler scanning. |
| [EventPattern](functions/EventPattern.mdx) | Marks a controller method as a **fire-and-forget** event handler. The method is called when a publisher emits an event matching `pattern`. No response is sent back to the publisher. |
| [MessageCtx](functions/MessageCtx.mdx) | Injects the microservice execution context into a method parameter. The context contains transport information and the matched pattern. |
| [MessagePattern](functions/MessagePattern.mdx) | Marks a controller method as a **request-response** message handler. The method is called when a client sends a message matching `pattern` and it is expected to return a response. |
| [Payload](functions/Payload.mdx) | Extracts the message payload (or a nested property of it) and injects it as a method parameter. |

## Interfaces

| Interface | Description |
| ------ | ------ |
| [CustomTransportStrategy](interfaces/CustomTransportStrategy.mdx) | Interface that custom transport strategies must implement. Pass an object conforming to this interface to connectMicroservice to use a transport not provided by this package. |
| [EventPatternMetadata](interfaces/EventPatternMetadata.mdx) | Shape of metadata stored by the [EventPattern](functions/EventPattern.mdx) decorator. |
| [IncomingEvent](interfaces/IncomingEvent.mdx) | An incoming event packet (fire-and-forget pattern). |
| [IncomingRequest](interfaces/IncomingRequest.mdx) | An incoming request packet (request-response pattern). |
| [MessagePatternMetadata](interfaces/MessagePatternMetadata.mdx) | Shape of metadata stored by the [MessagePattern](functions/MessagePattern.mdx) decorator. |
| [MicroserviceConfiguration](interfaces/MicroserviceConfiguration.mdx) | Base configuration shared by all built-in transport strategies. |
| [MicroserviceContext](interfaces/MicroserviceContext.mdx) | Contextual information injected via the [MessageCtx](functions/MessageCtx.mdx) decorator. |
| [MicroserviceExceptionContext](interfaces/MicroserviceExceptionContext.mdx) | Contextual information passed to exception filters when an error occurs inside a microservice message or event handler. |
| [MicroserviceServerInfo](interfaces/MicroserviceServerInfo.mdx) | Metadata stored per connected microservice. |
| [OutgoingRequest](interfaces/OutgoingRequest.mdx) | An outgoing request packet sent by a [ClientProxy](classes/ClientProxy.mdx). |
| [OutgoingResponse](interfaces/OutgoingResponse.mdx) | An outgoing response packet (request-response pattern). |
| [PatternHandler](interfaces/PatternHandler.mdx) | Metadata describing a registered pattern handler. |
| [RabbitMQOptions](interfaces/RabbitMQOptions.mdx) | Connection options for the RabbitMQ (amqplib) transport. |
| [RedisOptions](interfaces/RedisOptions.mdx) | Connection options for the Redis (ioredis) transport. |
| [Server](interfaces/Server.mdx) | Contract that every built-in transport server must satisfy. Custom transports only need to implement [CustomTransportStrategy](interfaces/CustomTransportStrategy.mdx). |
| [ServerFactoryInterface](interfaces/ServerFactoryInterface.mdx) | Factory interface for creating transport servers. |
| [TcpOptions](interfaces/TcpOptions.mdx) | Connection options for the TCP transport. |
